What it is

A structured-output prompting framework wrapped in a small web UI. Every model response is forced into a fixed schema — either a 5-slot pluralistic schema (LITERAL · MYTHIC · FUZZY · SIGIL · TWEET) for general use, or a 7-slot academic schema (CLAIM · EVIDENCE · UNCERTAINTY · IMPLICATION · OPEN · REFS · CONFIDENCE) for technical use — plus a self-reported confidence in [0, 1]. Format adherence is verified at parse time; non-conformant outputs fall back to a "raw" panel rather than being rendered as answers. The same UI runs against a local 4B model (Ollama gemma3:4b) and Cloudflare's free-tier 8B and 70B Llama models, allowing a small N-per-model study of format adherence and confidence calibration across a 17.5× parameter-count gap.

What is the finding — calibrated

Structured-output prompting is not new (Wei et al. 2022; OpenAI JSON-mode 2024; Anthropic tool-use schemas). The narrower observation here is counter-intuitive: format adherence on this 7-slot schema is non-monotonic in model scale. The smallest model (4B) and a mid-sized model (8B) both hit 12/12. The 70B drops to 8/12, with every failure attributable to omission of the same slot (IMPLICATION). This is consistent with the hypothesis that larger models carry stronger learned priors about answer structure that resist explicit format constraints. The prototype is open-source and the benchmark is one command to reproduce.

Honest limits

  • The pilot reported here is N=10 (5 questions × 2 providers). The expanded N=90 study (3 models × 30 questions) is included once the run completes — see the Stats Report. CIs and IQRs are reported there.
  • Self-reported confidence is not validated against ground-truth correctness. A calibration study against a labelled set is the obvious next step.
  • Smaller models occasionally violate the "do not fabricate citations" constraint. Inspection of REFS is the user's responsibility; the prompt is hardened but not infallible.
  • The interface cannot forecast events — neither model has post-cutoff knowledge or live web access. When asked for predictions, models say "consensus is silent" and lower confidence.
  • The schema is restrictive by design. It is a poor fit for narrative, code generation, or open-ended chat. Use general LLMs for those.
