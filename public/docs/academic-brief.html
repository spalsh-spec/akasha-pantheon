Larger LLMs are worse at honouring a 7-slot output schema than smaller ones — a small counter-intuitive finding.

A small empirical study of structured-output prompting across a 17.5× model-scale gap, with a working prototype and full reproducibility.

Sparsh Sharma · sparshsharma219@gmail.com · Melbourne · 10 May 2026

Empirical claim — and a partial counter-finding

A 7-slot prompt schema (CLAIM · EVIDENCE · UNCERTAINTY · IMPLICATION · OPEN · REFS · CONFIDENCE) enforced via a single system prompt achieves 100% format adherence (12/12) on a 4B local model and a 8B cloud model, but only 66.7% (8/12) on a 70B cloud model. All four 70B failures are attributable to the same omission: the model skips the IMPLICATION slot, treating it as redundant with the adjacent OPEN slot. The result is consistent with the hypothesis that larger models carry stronger learned priors about answer structure that resist explicit format directives — a result that inverts the naïve "bigger = more compliant" assumption and suggests the optimal model size for schema-enforced outputs may be smaller than expected.

Method

  1. Prompt: a single system prompt requires a one-sentence falsifiable CLAIM, supporting EVIDENCE with author-year citations and equations where available, an UNCERTAINTY section that is mandatory, an IMPLICATION, an OPEN-question framing falsification target, a numeric CONFIDENCE in [0,1], and a REFS line. The prompt explicitly forbids fabricated citations: when the model cannot cite, it must write "(unattributed consensus)" or "(needs verification)".
  2. Models:
    • 4B: gemma3:4b (Q4_K_M, Ollama, MacBook M1 16GB, local)
    • 8B: meta/llama-3.1-8b-instruct (Cloudflare Workers AI, free tier)
    • 70B: meta/llama-3.3-70b-instruct-fp8-fast (Cloudflare Workers AI, free tier)
  3. Questions: 12 prompts spanning physics & mathematics (4), complex-systems & biology (4), cognition / philosophy of science / comparative history (4). All within the model's training distribution; none requires post-cutoff knowledge.
  4. Scoring: format adherence (binary: did the response contain all 7 keys), self-reported confidence (parsed from the CONFIDENCE line), citation count (regex-detected author-year patterns), latency (wall-clock).

What is the finding — calibrated

Structured-output prompting is not new (Wei et al. 2022; OpenAI JSON mode 2024; Anthropic tool-use schemas). The narrower observation here is twofold:

  1. Format adherence is non-monotonic in model scale. 4B and 8B both hit 12/12 on the 7-slot schema. The 70B drops to 8/12. Wilson 95% CIs: [0.76, 1.00] vs [0.39, 0.86]. The intervals do not overlap.
  2. The failure mode is structured, not random. All four 70B omissions are the IMPLICATION slot specifically. The model evidently treats IMPLICATION as redundant with the adjacent OPEN slot and elides it. This pattern would be invisible in a JSON-mode benchmark that scores adherence binarily; it only surfaces because we record which field was skipped.

If replicated at larger N, this would suggest that schema-enforced output tasks may have an inverted-U relationship with model size — small models follow instructions because that is all they know how to do; large models override schemas with their own learned answer structure.

Honest limits

  • N=12 per model is small. The binomial CI on a 100%-adherence point estimate at N=12 is [73.5%, 100%] (Wilson score interval, 95%). Larger N is needed to tighten the bound; the methodology and harness are in place to scale it.
  • The benchmark was run on the same machine, in the same network conditions, in a single ~12-minute window. Latency numbers are point estimates, not steady-state load characterisation.
  • Self-reported confidence is not validated against ground-truth accuracy. A calibration study against a labelled answer set is the obvious next step.
  • Citation verification is the user's responsibility. The prompt forbids fabrication, but smaller models violate this constraint occasionally — sample inspection is required.
  • The 70B model occasionally skipped IMPLICATION (5/6 keys instead of 6/6) on the pilot. The expanded run will report this rate explicitly.

What I think this is useful for

Less as a generative tool, more as a thinking discipline: forcing a working scientist to produce CLAIM, EVIDENCE, UNCERTAINTY, OPEN before publishing or peer-reviewing is exactly the structure most readers want anyway. The interface enforces the structure; the model fills it; the human verifies citations. That last step is non-negotiable.
